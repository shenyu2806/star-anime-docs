# 管理后台

StarAnime 提供功能完备的管理后台，访问路径为 `/admin`，需要 **运营权限（role ≥ 2）** 才能进入。

## 访问控制

路由守卫中进行权限检查：

```js
if (to.meta.requiresOperator && role < 2) {
  next('/')   // 权限不足，跳转到首页
  return
}
if (to.meta.requiresAdmin && role < 3) {
  next('/admin')  // 需要管理员权限
  return
}
```

| 功能模块 | 最低权限 |
|----------|----------|
| 大部分管理功能 | 运营（role ≥ 2） |
| 系统日志 | 管理员（role = 3） |
| 系统设置 | 管理员（role = 3） |

## 管理后台布局

`AdminLayout.vue` 是管理后台的外层布局组件，提供：

- 左侧可折叠侧边栏（动态菜单，基于 `AdminNavConfig` 配置）
- 顶部操作栏（站点名称、用户信息、退出登录）
- 主内容区域（`<router-view>`）

## 功能模块一览

### 控制台（Dashboard）

`/admin` → `Dashboard.vue`

- 数据统计面板：用户总数、番剧总数、评论数、弹幕数
- 最近注册用户列表
- 最近新增番剧列表
- 系统状态概览

### 用户管理（UserList）

`/admin/users`

| 功能 | 说明 |
|------|------|
| 查看用户列表 | 分页、搜索用户名/邮箱 |
| 修改用户角色 | 将普通用户提升为博主/运营/管理员 |
| 封禁/解封账号 | 切换用户 `status` 字段 |
| 查看用户详情 | 头像、注册时间、最后登录时间 |

### 番剧管理（BangumiList）

`/admin/bangumi`

| 功能 | 说明 |
|------|------|
| 番剧 CRUD | 新增、编辑、删除番剧 |
| 封面管理 | 上传/替换番剧封面 |
| 剧集管理 | 在番剧下新增、编辑、排序剧集 |
| 可见性控制 | 切换番剧公开/隐藏状态 |

### 番剧资源管理（BangumiResource）

`/admin/resource`

| 功能 | 说明 |
|------|------|
| 查看资源列表 | 显示各剧集的视频资源及转码状态 |
| 删除资源 | 删除数据库记录并清理 HLS 切片文件 |
| 状态监控 | 查看转码进度（处理中/完成/失败） |

### 轮播图管理（BannerList）

`/admin/banner`

- 新增/编辑/删除首页轮播图
- 上传轮播图图片
- 设置轮播图链接、排序权重
- 切换显示/隐藏

### 公告管理（AnnouncementList）

`/admin/announcement`

- 新增/编辑/删除站点公告
- 设置公告级别（普通/重要/紧急）
- 控制公告显示时间范围
- 切换显示/隐藏

### 评论管理（CommentList）

`/admin/comments`

- 查看全站评论，支持搜索
- 隐藏/显示评论
- 批量删除违规评论

### 弹幕管理（DanmakuList）

`/admin/danmaku`

- 查看全站弹幕，支持按剧集筛选
- 隐藏违规弹幕
- 批量删除弹幕

### 审核功能

| 模块 | 路径 | 说明 |
|------|------|------|
| 番剧申请 | `/admin/apply` | 审核博主上传番剧资源的申请 |
| 博主申请 | `/admin/blogger-apply` | 审核普通用户升级为博主的申请 |
| 封号申请 | `/admin/ban-apply` | 处理用户举报（封号/解封请求） |
| 认证审核 | `/admin/certification` | 审核用户的个人/企业认证申请 |

### 用户评分管理（RatingList）

`/admin/ratings`

- 查看所有评分记录
- 删除异常评分

### 收藏管理（FavoriteList）

`/admin/favorites`

- 查看用户收藏记录
- 统计番剧收藏热度

### 观看记录管理（WatchHistoryList）

`/admin/watch-history`

- 查看用户观看历史
- 分析内容热度

### 系统日志（SystemLogs）

`/admin/logs`（需管理员权限）

后端通过 `logger` 中间件记录关键操作（登录、删除、权限变更等），日志存储在 `SystemLog` 表中。

- 按时间范围筛选
- 按操作类型筛选
- 按用户筛选
- 查看操作详情（请求参数、操作前后状态）

### 系统设置（SystemSettings）

`/admin/settings`（需管理员权限）

提供站点级别的配置管理，配置项存储在 `SystemConfig` 表：

| 配置项 | 说明 |
|--------|------|
| 站点名称 | 显示在页面标题和 Header |
| 站点副标题 | 显示在首页 Hero 区域 |
| 前台导航菜单 | 通过 `NavConfig` 动态配置 |
| 后台导航菜单 | 通过 `AdminNavConfig` 动态配置 |
| 注册开关 | 是否允许新用户注册 |
| 上传开关 | 是否允许上传视频 |

::: tip 导航配置
前台导航栏（Header）和管理后台侧边栏均支持通过系统设置动态配置，无需修改代码即可调整菜单结构。
:::
