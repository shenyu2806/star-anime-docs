CORE CODE · 01

JWT + Redis 认证机制

双重 Token 校验

采用 JWT 签名验证 + Redis 存储双保险，确保 Token 注销即时生效，防止 Token 盗用。

  • 登录成功后生成 JWT，同时写入 Redis（key: token:{userId}）
  • 每次请求先验证 JWT 签名，再比对 Redis 中存储的 Token
  • 退出登录时删除 Redis Key，原 Token 立即失效
  • 支持 TokenExpiredError 和 JsonWebTokenError 精细化错误处理
  • 用户 ID 挂载到 req.userId，下游 Controller 直接使用
middlewares/auth.js 
const jwt = require('jsonwebtoken')
const redis = require('../config/redis')

module.exports = async (req, res, next) => {
  try {
    const header = req.headers.authorization
    if (!header?.startsWith('Bearer '))
      return res.status(401).json({ message: '请先登录' })

    const token = header.split(' ')[1]

    // ① JWT 签名验证
    const decoded = jwt.verify(token, process.env.JWT_SECRET)

    // ② Redis 存活校验（防注销后复用）
    const stored = await redis.get(`token:${decoded.userId}`)
    if (!stored || stored !== token)
      return res.status(401).json({ message: 'Token已失效' })

    req.userId = decoded.userId
    next()
  } catch (err) {
    if (err.name === 'TokenExpiredError')
      return res.status(401).json({ message: 'Token已过期' })
    res.status(401).json({ message: '无效Token' })
  }
}
CORE CODE · 02

Redis 缓存加速策略

番剧详情缓存

番剧详情是高频读取、低频变更的数据，通过 Redis 缓存降低数据库压力，提升响应速度。

  • 缓存 Key 格式：bangumi:detail:{id}，TTL 3600 秒
  • 每次请求先查 Redis，命中则直接返回，跳过 MySQL 查询
  • 缓存未命中时查库，并将结果序列化写入 Redis
  • 同时对播放量 views 执行原子 increment 操作，精准统计
  • 编辑番剧时需主动删除对应缓存键，保证数据一致性
controllers/bangumiController.js 
// 获取番剧详情（Redis 缓存）
exports.getDetail = async (req, res) => {
  try {
    const { id } = req.params
    const cacheKey = `bangumi:detail:${id}`

    // ① 优先读取 Redis 缓存
    const cached = await redis.get(cacheKey)
    if (cached) {
      return res.json({
        code: 200,
        data: JSON.parse(cached)
      })
    }

    // ② 缓存未命中，查 MySQL
    const bangumi = await Bangumi.findOne({
      where: { id, status: 1 }
    })
    if (!bangumi)
      return res.status(404).json({ message: '番剧不存在' })

    // ③ 播放量 +1（原子操作）
    await bangumi.increment('views')

    // ④ 写入缓存，TTL = 1小时
    await redis.setEx(cacheKey, 3600,
      JSON.stringify(bangumi))

    res.json({ code: 200, data: bangumi })
  } catch (err) { ... }
}

弹幕系统实现

实时弹幕发送

弹幕按视频时间轴精确绑定到剧集，支持三种展现类型和自定义颜色，完整的校验保障内容安全。

  • 弹幕绑定 episode_id + time（秒级精度），按时间升序返回
  • 类型：0-滚动 / 1-顶部固定 / 2-底部固定
  • 支持 hex 颜色值自定义，默认白色 #ffffff
  • 内容限 100 字，服务端严格校验非空、长度、时间合法性
  • 获取弹幕无需登录，发送弹幕需 JWT 认证

controllers/danmakuController.js 
// 发送弹幕（需登录）
exports.send = async (req, res) => {
  const userId = req.userId
  const {
    episode_id, time, text,
    color = '#ffffff', type = 0
  } = req.body

  // ① 参数校验
  if (!text?.trim())
    return res.status(400).json({ msg: '内容不能为空' })
  if (text.trim().length > 100)
    return res.status(400).json({ msg: '不能超过100字' })
  if (![0,1,2].includes(Number(type)))
    return res.status(400).json({ msg: '类型无效' })

  // ② 写入数据库
  const dm = await Danmaku.create({
    episode_id, user_id: userId,
    time: parseFloat(time),
    text: text.trim(),
    color, type: Number(type)
  })

  res.json({ code: 200, message: '发送成功',
    data: { id: dm.id, time: dm.time,
             text: dm.text, color: dm.color }
  })
}
CORE CODE · 04

HLS 多分辨率视频流

自适应视频流检测

系统自动扫描 HLS 文件目录，构建多分辨率清单，优先返回最高清晰度，无需硬编码配置。

  • 服务端扫描 uploads/hls/{episodeId}/ 目录
  • 自动识别 1080p / 720p / 480p / 360p 各分辨率 m3u8
  • 优先使用 master.m3u8，无则用最高分辨率代替
  • 支持数据库存储的多分辨率 video_urls JSON 字段
  • 前端 ArtPlayer + HLS.js 无缝接入，支持实时分辨率切换
目录结构
uploads/hls/{episodeId}/
├── master.m3u8
├── 1080p.m3u8
├── 720p.m3u8
├── 480p.m3u8
└── *.ts (分片文件)
controllers/bangumiController.js 
const QUALITY_ORDER = ['1080p','720p','480p','360p']

function detectHLS(episodeId) {
  const dir = path.join(HLS_ROOT, String(episodeId))
  if (!fs.existsSync(dir))
    return { hls_url: null, qualities: [] }

  // 扫描各分辨率 m3u8 文件
  const qualities = []
  for (const label of QUALITY_ORDER) {
    const m3u8 = path.join(dir, `${label}.m3u8`)
    if (fs.existsSync(m3u8)) {
      qualities.push({
        label: label.replace('p', 'P'),
        url: `/uploads/hls/${episodeId}/${label}.m3u8`,
        default: qualities.length === 0
      })
    }
  }
  if (qualities.length === 0)
    return { hls_url: null, qualities: [] }

  const hasMaster = fs.existsSync(
    path.join(dir, 'master.m3u8'))
  return {
    hls_url: hasMaster
      ? `/uploads/hls/${episodeId}/master.m3u8`
      : qualities[0].url,
    qualities
  }
}
